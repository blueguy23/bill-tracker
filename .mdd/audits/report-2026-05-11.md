# MDD Audit Report — 2026-05-11

## Executive Summary

Full audit of 9 features across 60+ source files. Found **45 issues**: 5 critical, 13 high, 15 medium, 12 low. The most impactful bugs are sign-convention errors in credit health alerts (always show $0 paydown) and missing validation in subscription anchoring and credit settings. Test coverage is strong for business logic engines but weak for API routes and mutation handlers.

## Findings by Severity

### CRITICAL (5)

1. **[03] Historical import has no quota guard** — `runHistoricalImport` can blow past `SIMPLEFIN_QUOTA_GUARD` with 3 fetches, no pre-check
2. **[03] `markTransfersById` sequential awaits** — partial failure leaves transfers in inconsistent state, violates project rules
3. **[04] Budget notification uses wrong metric** — passes `spent` instead of `linearProjectedTotal`, creating split-brain between UI status and alert trigger
4. **[07] Missing `lastCharged` validation** — `handleAnchorSubscription` produces NaN dueDate when field is missing
5. **[08] Credit settings has no field-level validation** — accepts day 999, negative limits, arbitrary values written to DB

### HIGH (13)

1. **[03] Category rules fetched per-transaction** — 300 DB reads per sync instead of 1
2. **[04] Rollover has no idempotency guard** — double-run compounds carry-forward
3. **[04] Budget notification fire-and-forget** — upstream DB failures produce unhandled rejections
4. **[05] Recent Payments shows purchases** — queries `amount < 0` (charges) instead of `amount > 0` (payments)
5. **[05] `paydownTo30Pct` sign error** — negative balance arithmetic always returns $0
6. **[05] `paydownToTarget` sign error** — statement close alerts always show $0 paydown
7. **[06] GET endpoints with side effects** — test and digest notification routes mutate state via GET
8. **[06] Sync notification never deduplicates** — `Date.now()` key is always unique, no cooldown entry
9. **[07] `dismissSubscription` swallows all errors** — masks DB failures as success
10. **[07] Side effects on GET** — `handleListSubscriptions` writes charge history on every page load
11. **[08] Utilization trend sign logic** — balance reconstruction may underreport historical utilization
12. **[09] Amortize removes from UI without bill** — subscription reappears on next load
13. **[09] Duplicate `isTransfer` logic** — `TransactionsView` local function diverged from canonical stored field

### MEDIUM (15)

1. **[01] Recurring bill double-counting** — `isPaid` is not per-month scoped
2. **[01] Silent fetch error swallowing** — `.catch(() => {})` discards API errors
3. **[02] No UUID format validation** — billId path param goes straight to DB
4. **[02] Orphaned payments** — bill deletion doesn't clean up payment records
5. **[03] `RawSFINAccount.org` typed as required** — should be optional per real API behavior
6. **[03] `incrementUrlUnits` silent no-op** — no error when sync log missing
7. **[03] Cron parser only handles simple format** — non-standard cron values produce wrong next-sync time
8. **[05] Raw negative balance display** — CreditAccountCard shows negative dollar amount
9. **[06] Wrong error reason code** — webhook send failure returns `'no_webhook'`
10. **[07] `inferCategory` defaults to `'subscriptions'`** — miscategorizes non-subscription recurring charges
11. **[07] Substring match has no minimum length** — single-char bill names match everything
12. **[07] Dismissed subscriptions hard cap** — 1000 limit with no pagination
13. **[08] UTC/local timezone mismatch** — statement close day comparison
14. **[08] `credit/page.tsx` violates Rule 10** — calls handlers returning NextResponse instead of direct adapters
15. **[09] Amortization always divides by 12** — incorrect for quarterly/weekly subscriptions

### LOW (12)

Various: dead code (CreditView component), missing pagination signals, build-time env var divergence, Infinity validation gap, no tests for amortization, fire-and-forget post-sync handlers.
